Class data.core.GameObject

The objects that represent anything in a stage.

Players, enemies, powerups, stage artifacts… all of them should be GameObjects. GameObjects are classless objects, so there is no class inheritance. Instead, you should manipulate the GameObject’s components. Also, do not dynamically add methods to a GameObject, as it may interfere with the callback mechanism.

Methods

Methods
GameObject:addComponent (newComponent)
Adds a component to this GameObject. The component may add itself as one of the GameObject’s fields. Be sure to not add more than one component of each type, as one will only partially overwrite the other. Future releases might include the removal of components.

Parameters:

  • newComponent Component
GameObject:register (event, component)
Registers a component to be called upon a specified event. The method event of component will be called whenever the event method is called on this GameObject.

Parameters:

  • event string The event name
  • component Component The component to register.
GameObject:setEventHandler (event, handler)
Sets a global handler for event. The handler will be called before the registered objects.

Parameters:

  • event string The event to register
  • handler function The global handler to set.
GameObject.new ()
Creates and returns a new GameObject. Strings that cannot be registered as events are: addComponent, register and setEventHandler.

Returns:

    GameObject The newly created GameObject.
