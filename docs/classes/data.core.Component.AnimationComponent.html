Class data.core.Component.AnimationComponent

A Component that represents an element’s sprites and animations.

Building and destroying

AnimationComponent.__call (spriteData, spriteSize[, spriteOffset])
Builds a new AnimationComponent.

Parameters:

  • spriteData string or Image The component’s sprites. Can be supplied as a file path (as a string) or as an already loaded (using love.graphics.newImage) sprite sheet.
  • spriteSize vector The sprite size for the animation in pixels.
  • spriteOffset vector The sprite offset for the animation in pixels. Default is no offset.

Returns:

    AnimationComponent The newly created AnimationComponent.
AnimationComponent:addTo (container)
Adds this component to a GameObject. This method registers the update, changeToState and draw methods with the container GameObject. The AnimationComponent will be added as the animation field of the GameObject.

Parameters:

  • container GameObject The GameObject this component is being added to.

Drawing and animation data

AnimationComponent:draw ()
Draws the AnimationComponent’s current animation. The default positioning is such that the animation’s bottom-center overlaps the container GameObject’s position.
AnimationComponent:setSpriteData (spriteData, spriteSize[, spriteOffset])
Sets the AnimationComponent’s sprite data for it to be drawn. This method is called when the Component is initialized, so you should only call it if you want to alter sprite data dynamically.

Parameters:

  • spriteData string or Image The component’s sprites. Can be supplied as a file path (as a string) or as an already loaded (using love.graphics.newImage) sprite sheet.
  • spriteSize vector The sprite size for the animation in pixels.
  • spriteOffset vector The sprite offset for the animation in pixels. Default is no offset.
AnimationComponent:addAnimation (name, params)
Adds an animation to the component.

Parameters:

  • name string The name of the new animation. If the component already had an animation with this name, it will be overwritten.
  • params table A table of animation parameters. Mandatory entries in this table are “mode” (animation mode, which can be “loop”, “once” or “bounce”); frames, which must be an array of frame coordinates as strings; and defaultDelay, the default time for each animation frame. Optional entries are delays, an array of numbers representing each frame’s duration; flippedH and flippedV, the animation’s horizontal and vertical flipping respectively.
AnimationComponent:setAnimation (name[, noReset=false])
Sets the component’s current animation.

Parameters:

  • name string The name of the animation to set. If no animation with such name exists, an error is raised.
  • noReset bool A flag to prevent restarting the animation if true. (default false)

State handling

AnimationComponent:changeToState (nextState)
Executes a transition to a specified state. The next state’s animation is set as the current animation, if it differs from the current one.

Parameters:

  • nextState State The target state.
AnimationComponent:update (dt)
Updates the Component, advancing the animation.

Parameters:

  • dt number The time interval to apply to the Component.
