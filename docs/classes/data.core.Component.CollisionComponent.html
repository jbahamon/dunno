CollisionComponent:addTo (container)
Adds this component to a GameObject. This method registers the move, moveTo, draw, changeToState, destroySelf and getHitBy methods with the container GameObject. The CollisionComponent will be added as the collision field of the GameObject.

Parameters:

  • container GameObject The GameObject this component is being added to.

State handling

Called when the container GameObject changes state. This method changes the component’s collision box to the target state’s collision box if it differs from the current one.

Parameters:

Collision handling

CollisionComponent:setColliders (tileCollider, activeCollider)
Sets the colliders for this CollisionComponent, adding the current collision box to them.
If the Component already had colliders, it is removed from them.

Parameters:

  • tileCollider TileCollider The tileCollider to set.
  • activeCollider HardonCollider The activeCollider to set.
CollisionComponent:movementIntoCollidingBox (box)
Calculates the displacement needed to move the CollisionComponent into a colliding box. The movement is performed in the axis with the smallest distance to the target box. Useful for room transitions, for example.

Parameters:

  • box Shape The collision box to move the CollisionComponent into.

Returns:

    number The needed displacement that should be applied.
CollisionComponent:resetCollisionFlags ()
Resets the CollisionComponent’s collision flags.
CollisionComponent:onTileCollide (dt, tileCollisionComponent, tile, position)
Called when colliding with a tile from the stage’s collision layer.
Essentially, it adds the collision event to the pendingCollisions field. This function should not be overriden; if you want to implement your own collision resolution for tiles, override CollisionComponent:resolveTileCollisions.

Parameters:

  • dt number The time slice for the collision frame.
  • tileCollisionComponent Tile A sample tile that can be used to recreate the collision (should be removed).
  • tile Tile The tile CollisionComponent with which the CollisionComponent is colliding.
  • position vector The position of the colliding tile, measured in tiles.

See also:

  • CollisionComponent:resolveTileCollisions
  • CollisionComponent:resetCollisionFlags
CollisionComponent:resolveTileCollisions (sampleTile, tileSize)
Resolves collisions with tiles. Iterates over the registered collision events and resolves them appropriately. This function can be overriden to implement custom tile collision resolution, but be careful to treat every tile type.

Parameters:

  • sampleTile Tile A tile of the appropriate size that can be moved around to simulate the collisions.
  • tileSize vector A vector that contains the size of the stage’s tiles, in pixels.

See also:

  • CollisionComponent:onTileCollide
  • CollisionComponent:resetCollisionFlags
CollisionComponent:onDynamicCollide (dt, otherCollisionComponent)
Called when colliding with an active CollisionComponent in the world (interactive CollisionComponent, enemy, etc). Currently calls getHitBy on the other CollisionComponent’s container if this CollisionComponent damages on contact.

Parameters:

  • dt number The time slice for the collision frame.
  • otherCollisionComponent CollisionComponent The CollisionComponent that is colliding with this one.
CollisionComponent:getHitBy (otherObject)
Called when hit by something (read: something that does damage)– Currently sets the hit flag in the collisionFlags field.

Parameters:

  • otherObject GameObject The GameObject hitting this CollisionComponent
CollisionComponent:setCollisionBox (newSize)
Sets the current collision box for the CollisionComponent.

Parameters:

  • newSize vector The size of the new collision box in pixels.
