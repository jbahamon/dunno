Class data.core.GameObjectFactory

A class for mass producing GameObjects without loading every resource each time.

In particular, both sprite data and the parameter file are loaded a single time. Useful for respawning enemies or other GameObjects (e.g. bullets).

GameObjectFactory._call (parameters, The, activeCollider, folder)
Builds a new GameObjectFactory.

Parameters:

  • parameters table The parameters to use in the building of GameObjects. It must include a sprite field, with sheet, spriteSizeX and spriteSizeY as fields. The sheet must be an image loaded with love.graphics.newImage.—
  • The TileCollidertileCollider TileCollider to assign to every GameObject created by this GameObjectFactory.
  • activeCollider HardonCollider The HardonCollider instance to assign to every GameObject created by this GameObjectFactory.
  • folder string The base folder to load files (sprites, classes, etc) for building the GameObjects from.

Returns:

    GameObjectFactory The newly created GameObjectFactory.

GameObjectFactory:createAt (position, facing)
Creates a new instance of the GameObjectFactory’s GameObject at the specified position and facing the specified direction.

Parameters:

  • position vector The position, in pixels, where to put the new GameObject.
  • facing number The direction the GameObject should be facing. A value of facing greater than zero indicates facing right; a value smaller than zero indicates facing left.

Returns:

    GameObject The newly created GameObject.
