Module Element

An element implementation. Extends StateMachine. Every dynamic element of a stage (players, enemies, interactive objects) is an Element or inherits from this class. Elements are currently animated using anim8 s and have a collision box, a HardonCollider shape. An Element's position is considered to be its feet's position (bottom-center).

Element:addState (state)
Adds a state to the Element. A state can only belong to a single machine at a time: it should not belong to another state machine when this method is called. Call removeState on the other machine first. If the new state has a collision box, its information is added to the Element's collider.

Parameters:

  • state: The state to be added.

Element:changeToState (nextState)
Executes a transition to a specified state. The current state's onExitTo and the target state's onEnterFrom are executed, if found. The Element's collision box is adjusted if the next state has a different box from the current one.

Parameters:

  • nextState: The target state.
Element:checkStateChange ()
Checks for conditions and executes any possible state change. If two or more transitions are possible, the one with higher priority is taken (or the one that was added first, if there is more than one transition with the sane priority).
Element conditions can use an additional parameter: the collision flags.

Element:onTileCollide (dt, tileElement, tile, x, y)
Called when colliding with a tile from the stage's collision layer. Essentially, adds the collision event to the pendingCollisions field. This function should not be overriden; if you want to implement your own collision resolution for tiles, override Element:resolveTileCollisions.

Parameters:

  • dt: The time slice for the collision frame.
  • tileElement: The tile Element with which the Element is colliding.
  • tile:
  • x: the horizontal position of the colliding tile, measured in tiles
  • y: the vertical position of the colliding tile, measured in tiles

See also:

  • Element:resolveTileCollisions
  • Element:resetCollisionFlags
Element:removeState (stateName)
Removes a state from the machine, leaving it with no owner. If the state had a collision box, it will be removed from the colliders. Therefore, it is NOT recommended to use the same collision box in different Elements: use copies instead.

Parameters:

  • stateName: The name of the state to be removed. If there is no state with such name in the machine, nothing is done.
Element:resetCollisionFlags ()
Resets the Element's collision flags.
Element:resolveTileCollisions (sampleTile, tileSize)
Resolves collisions with tiles. Iterates over the registered collision events and resolves them appropriately. This function can be overriden to implement custom tile collision resolution.

Parameters:

  • sampleTile: A tile of the appropriate size that can be moved around to simulate the collisions.
  • tileSize: A vector that contains the size of the stage's tiles, in pixels.

See also:

  • Element:onTileCollide
  • Element:resetCollisionFlags
