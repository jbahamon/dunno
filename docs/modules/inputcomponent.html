Module inputcomponent

A Component that represents keyboard input for a GameObject.

Building and destroying

InputComponent:init (bindings) Builds a new InputComponent with the given bindings, or default ones.
InputComponent:addTo (container) Adds this component to a GameObject.
InputComponent:setBindings (bindings) Sets the player's bindings.

State handling

InputComponent:update (dt) Updates the InputComponent, applying control physics if necessary.
InputComponent:setControl (control) Sets the control flag for the Component.


Building and destroying

InputComponent:init (bindings)
Builds a new InputComponent with the given bindings, or default ones. The default bindings are as follows: WASD and arrows can be used as the direction buttons; Space and Z can be used as the jump button, and Right Control and X can be used as the special/attack button.

Parameters:

  • bindings The key bindings to set, as a table. See TLbind's documentation for details (under bind.keys).
InputComponent:addTo (container)
Adds this component to a GameObject. This method registers the update method with the container GameObject. The InputComponent will be added as the input field of the GameObject.

Parameters:

  • container The GameObject this component is being added to.
InputComponent:setBindings (bindings)
Sets the player's bindings.

Parameters:

  • bindings The bindings to set, as a table. See TLbind's documentation for details (under bind.keys).

State handling

InputComponent:update (dt)
Updates the InputComponent, applying control physics if necessary.

Parameters:

  • dt The time interval to apply to the Component.
InputComponent:setControl (control)
Sets the control flag for the Component. A value of false will disable input.

Parameters:

  • control The value for the control flag.
