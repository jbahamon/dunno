Class ElementState

A StateMachine's state implementation. Extends State. It is the base for all other states.

Positioning and dynamics
ElementState:move (dx, dy)
Moves the Element by a certain amount, in pixels. Called by Element:move. The ElementState's collision box, if any, is not moved.

Parameters:

  • dx The horizontal displacement.
  • dy The vertical displacement.
ElementState:moveTo (x, y)
Moves the Element to a certain position, in pixels. Called by Element:moveTo. The ElementState's collision box, if any, is not moved.

Parameters:

  • x The horizontal position.
  • y The vertical position.
ElementState:turn ()
Flips the Element horizontally, reversing its velocity. Note that a positive horizontal velocity always means moving towards the right.
ElementState:applyFriction (dt, frictionForce)
Applies the provided friction force to the Element. A friction force is always dissipative: it will never cause the element to reverse its velocity. This means that every Element has the same mass, for the time being.

Parameters:

  • dt The current frame's time slice, in seconds
  • frictionForce The friction to apply, in acceleration units (pixels/seconds^2)
ElementState:stepDynamics (dt, acceleration)
Applies an acceleration to the Element and moves it. It calls ElementState:applyPostForceEffects to apply those effects that come into play after applying a force but before moving the Element (such as friction). This function should not be called alone: instead, it is called from ElementState:update.

Parameters:

  • dt The current frame's time slice, in seconds.
  • acceleration The acceleration to be applied.
ElementState:applyPostForceEffects (dt)
Applies those effects that come into play after applying a force but before moving the Element (such as friction). The only default effect is applying friction.

Parameters:

  • dt The current frame's time slice, in seconds.

see also:

    ElementState:applyFriction
ElementState:getCurrentAcceleration (dt)
Returns the acceleration to be applied to the Element. The default implementation returns the default acceleration values specified in the dynamics table.

Parameters:

  • dt The current frame's time slice, in seconds.

ElementState:onEnterFrom (previousState)
Initializes the ElementState with info (position, velocity, facing, etc) from the previous one. It also starts the ElementState's animation from its first frame. Automatically called from StateMachine:changeToState.

Parameters:
