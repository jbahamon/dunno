Class Player

A game player implementation. Extends Element. A player has all of the Element's features, plus receiving input from the user.

Player:addBasicStates (basicStatesParams)

Adds the basic states and transitions for a player. Called by Player:loadBasicStates. The table entries must have the following structure:

  • animationData: An anim8 animation.

  • dynamics: A dynamics table, as specified (TODO :( )

Parameters:

  • basicStatesParams The basic states' parameters, as a table. The table must have "stand", "walk", "jump", "fall" and "climb" entries.
Player:getDefaultStateClass ()
Returns the Player's default state class (a hump class). This method is used when building a Player from a file, to determine the class used when no state class is specified. For a Player, it's PlayerState; override this method if you want to create a character with a custom base state.

Returns:

    The hump class to be used in the construction of this Player's states when no class is specified.
Player:loadBasicStates (parameters, folder)
Adds a Players' basic states and transitions from a parameter table. Note that calling this function is completely optional, but offers a way to build the common core of many platforming characters. You can avoid using this function and building al of a Player's states and transitions from scratch

Parameters:

  • parameters The parameter table, as described in (TODO :c)
  • folder (Optional) the specific folder to load the state from. If omitted, the Player's folder is used.
Player:loadBasicFromParams (parameters, folder)
Loads an empty Player from a minimal set of parameters.

Parameters:

  • parameters The set of parameters to be used. It must have width and height as fields.
  • folder The base folder to load files (sprites, classes, etc) from. Required (in opposition to uses of folder parameters in Player methods).
